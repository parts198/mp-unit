# swaga

В репозитории размещено одностраничное веб-приложение, которое подтягивает данные по товарам из API OZON, рассчитывает показатели, эквивалентные формулам в Excel, и отображает всё в интерактивной таблице.

## Как пользоваться

1. Скопируйте `stores.sample.js` в `stores.secrets.js` и впишите реальные пары `client_id`/`api_key` для своих магазинов. Файл `stores.secrets.js` занесён в `.gitignore`, поэтому ключи не попадут в репозиторий; без него страницы покажут предупреждение и список магазинов будет пустым.
2. Для страницы `sync.html` можно заранее скачать прайсы поставщика сервером: выполните `node scripts/fetch-prices.js`. Файлы сохранятся в `prices/<clientId>_<warehouseId>.xlsx` и будут использоваться, если прямой запрос из браузера заблокирован CORS.
2. Откройте `index.html` в браузере (достаточно двойного клика или запуска через любой статический сервер). Переключаться между ценами, акциями и заказами можно через кнопки «Цены» / «Акции» / «Заказы» в правой части шапки.
3. В верхнем меню выберите магазин и задайте нужный фильтр по артикулу (учитываются все `offer_id`, даже те, что находятся на других листах).
4. Задайте объём листа в выпадающем списке «Товаров на листе» (100, 500, 1000 или 5000 позиций). При перелистывании счётчик строк и поиск работают по всему массиву товаров.
5. Нажмите **«Обновить данные»**. Приложение постранично выгружает **все** `offer_id` магазина через `/v3/product/list`, затем чанками запрашивает `/v5/product/info/prices` и конструирует строки таблицы.
6. В таблице редактируемы только:
   - **Цена** — при изменении автоматически пересчитывается эквайринг как 1,9 % от новой цены.
   - **Себестоимость** — участвует во всех формулах выплат, маржи и маржинальности.
   - **Наценка** — при вводе целевого процента (от себестоимости) таблица пересчитывает цену, комиссии и остальные показатели.
7. Остальные поля защищены от правок и всегда рассчитываются по формулам Excel: комиссия в рублях, затраты на FBS (комиссия + логистика + доставка + первая миля + упаковка + продвижение + эквайринг), выплаты селлеру, маржа и маржинальность. Новые столбцы **«Упаковка»** (фиксированные 40 ₽) и **«Продвижение»** (1 % от текущей цены) автоматически учитываются в расходах, поэтому при изменении цены или наценки значения пересчитываются.
8. Чтобы сузить выдачу, используйте строку «Фильтр по артикулу» в шапке — отображаются только те строки, где `offer_id` содержит введённый фрагмент, причём фильтрация идёт по всей выборке, а не только по текущему листу.
9. В блоке «Наценка для всех» можно задать процент и применить его ко всем товарам сразу — цены, эквайринг и остальные метрики пересчитаются автоматически. Рядом задаётся «Наценка для минимальной цены» — её процент используется для расчёта `min_price`, который уходит в OZON вместе с основной ценой.
10. При отправке цен в OZON (`/v1/product/import/prices`) значения `price` и `min_price` отправляются строками; `vat` выставляется автоматически (0.2, кроме магазина с `client_id` 2521951, где используется 0). Кнопки отправки позволяют обновить выбранные строки (отмеченные галочками в таблице) или весь ассортимент.
11. Интерфейс адаптируется к тёмным темам и мобильным устройствам: все поля ввода остаются читаемыми, а таблица превращается в карточки со всплывающими подписями для узких экранов.
12. Страница `actions.html` читает XLSX (кнопка «Выбрать XLSX») и дополнительно тянет комиссии/себестоимости **и актуальные цены (current/min_price)** через `/v5/product/info/prices`, чтобы показать «Наценку», «Маржу» и обновить столбцы «Текущая цена»/«Минимальная цена» так же, как на странице цен. Если файл уже загружен, а магазин переключён, нужные данные подтянутся автоматически; повторные идентичные запросы к OZON берутся из кэша, чтобы не гонять одни и те же данные. Файлы и рассчитанные строки сохраняются в локальном «серверном» хранилище с датой из свойств файла (используется фактическое время создания XLSX или timestamp файла) и SHA‑256 хэшем, поэтому одинаковые выгрузки с сегодняшней датой автоматически пропускаются.
13. Если в файле несколько листов, интерфейс автоматически берёт тот, где после заголовка «Артикул» больше всего строк, и показывает имя листа и количество строк в статусе и логе.
14. В таблицу попадают столбцы «Артикул», «Название», «Ваша цена», «Текущая цена», «Минимальная цена», «Участие товара в акции» (чекбокс), «Итоговая цена по акции» (редактируемая), «Акционный бустинг в поиске», «Наценка» (в процентах, как на странице цен), «Маржа» (в рублях, как на странице цен), «Цена для минимального/максимального акционного бустинга». Скрытые столбцы из XLSX тоже читаются и участвуют в формулах.
15. Чекбоксы участия изначально пустые и появляются только после ручных правок или автозаполнения. «Акционный бустинг в поиске» пересчитывается по формуле из файла (та же, что в Excel) и показывается в процентах со знаком «+» и зелёным градиентом (пример: `+46,4%`). «Наценка» остаётся процентной, «Маржа» показывается в рублях; отрицательные значения подсвечиваются красным, от 0 до 40 % действует зелёный градиент. Галочки автоматически включаются при любых изменениях, если выполнены условия: бустинг > 0, есть себестоимость, минимальная цена не выше порога бустинга и наценка не падает ниже 15 %. При нарушении условий чекбоксы снимаются автоматически.
16. Для каждого магазина выбранный файл и рассчитанные показатели хранятся отдельно: переключите магазин и загрузите свой XLSX, а статус подскажет, сколько строк прочитано. Файлы сохраняются в локальном «серверном» хранилище с датой и SHA‑256 хэшем, поэтому идентичные выгрузки с сегодняшней датой автоматически пропускаются. Блок «Лог действий» фиксирует загрузки, ошибки чтения и ответы API; прогресс‑бар под заголовком показывает ход запросов.
17. Можно выбрать сразу несколько XLSX: система читает колонку **OzonID** и автоматически сопоставляет файл с магазином, в каталоге которого совпадают product_id. Благодаря этому подойдут файлы с любыми именами (`products-1977747`, `products-1977747 (1)` и т. п.) — главное, чтобы в столбце OzonID были идентификаторы из нужного магазина. Если включён режим «Только выбранный магазин», файл с нулём совпадений OzonID будет отклонён с сообщением об ошибке, чтобы не подмешивать данные чужого магазина.
18. Чтобы ускорить сопоставление, используйте чекбокс **«Только выбранный магазин»** рядом со списком магазинов — тогда поиск OzonID в файлах будет выполняться только по выбранной витрине (без перебора остальных), что заметно снижает время загрузки. Для очень больших файлов таблица рендерится порциями, чтобы страница оставалась отзывчивой.
19. Кнопка **«Выставить итоговые цены»** ставит «Итоговую цену по акции, RUB» по трём шагам: если ваша минимальная цена выше потолка бустинга — подставляется «Цена для максимального акционного бустинга»; если минимальная цена попадает внутрь коридора между «Ценой для минимального акционного бустинга» и «Ценой для максимального акционного бустинга» (учитывается любой порядок значений) — подставляется минимальная цена; если минимальная цена ниже этого коридора, поле остаётся пустым. Кнопка **«Итог = текущая цена»** заполняет итоговую цену из столбца «Текущая цена, RUB» и сразу включает участие для строк с валидными данными. Чекбокс участия включается только когда есть себестоимость, бустинг больше нуля и расчёт маржи не уходит в минус. Над таблицей прогресс‑бар показывает долю строк с итоговой ценой, а второй прогресс‑бар ведёт выполнение сетевых запросов; лог действий свёрнут по умолчанию и находится под отдельным прогресс‑баром. Для массового выбора добавлен мастер‑чекбокс в заголовке столбца «Участие товара в акции», который выделяет/снимает все видимые строки с валидной ценой и бустингом.
20. Кнопка **«Показать только изменённые»** оставляет в таблице строки, где поменялись цена или участие (удобно для массового обновления), а кнопки добавления/удаления товаров работают только по отмеченным чекбоксам или по всем строкам с ценой.
21. В блоке действий добавлен ввод **ID акции** и кнопки «Добавить отмеченные», «Добавить все с ценой», «Удалить отмеченные» и «Удалить все, что уже в акции». Первые две используют `/v1/actions/products/activate`, удаление отмеченных и массовое удаление — `/v1/actions/products/deactivate`. Перед удалением всех система запрашивает список текущих участников через `/v1/actions/products`, поэтому убрать можно одним кликом и уже находящиеся в акции товары.

22. Страница `orders.html` теперь грузит одновременно FBS и FBO отправления за заданный диапазон дат: запрашивает `/v3/posting/fbs/list` и `/v2/posting/fbo/list`, складывает ответ в локальный кэш и при повторном запросе дозагружает только новые записи внутри того же диапазона. Для всех `offer_id` подтягиваются комиссии и себестоимости через `/v5/product/info/prices`, а для FBO товаров запрашивается среднее время доставки через `/v1/analytics/average-delivery-time/summary` (кешируется), чтобы рассчитать логистику по тарифам FBO: базовый объёмный тариф + коэффициент/процент от цены, после чего результат ограничивается границами `fbo_direct_flow_trans_min_amount`/`fbo_direct_flow_trans_max_amount` и добавляется последняя миля `fbo_deliv_to_customer_amount`. В таблице появилась колонка «Схема» (FBS/FBO) и фильтр по схеме; выручка считается как на странице цен (цена минус эквайринг 1,9 %, продвижение 1 %, доставка/логистика/первая миля/упаковка 40 ₽ для FBS; для FBO — эквайринг, продвижение, логистика FBO без отдельной упаковки + комиссия FBO), маржа — разница выручки и себестоимости (на количество), наценка = (выплата − себестоимость)/себестоимость. Строки показывают номер заказа, дату, схему, статус, сумму, выручку, маржу, маржинальность, наценку, признак начислений и перечень позиций; мини-таблица с расходами по позициям скрыта по умолчанию и раскрывается кнопкой «Показать» в колонке «Расходы», верхний итог агрегирует все значения, а лог запросов фиксирует каждый вызов OZON для отладки. Селектор магазинов принимает как массив `[{name, client_id, api_key}]`, так и объект `{stores:[...]}` и остаётся доступным даже без `stores.secrets.js` (используется `stores.sample.js` или встроенный демо-магазин). В колонке «Начисления» ссылка открывает `transactions.html` с уже заполненным магазином и фильтром по отправлению, чтобы сразу перейти к нужной выплате.

23. Комиссии, логистика и себестоимости из `/v5/product/info/prices` фиксируются в локальном кэше по артикулу и магазину при первом получении, чтобы при последующих пересчётах использовать те же исходные данные (на случай, если тарифы на стороне OZON поменяются). Аналогично начисления (`/v3/finance/transaction/list`) сохраняются в `localStorage` и подтягиваются в таблицу заказов: если по отправлению есть операции, в колонке «Начисления» появляется значок 💰 и расчёты выручки/маржи используют фактическую сумму.

24. Добавлена отдельная страница `transactions.html` для работы с начислениями: задайте магазин и период (не больше месяца), нажмите «Загрузить начисления» — данные подтянутся постранично через `/v3/finance/transaction/list`, сохранятся в кэше (если браузер позволяет) и будут сгруппированы по отправлениям с раскрываемыми деталями и фильтром по номеру. Если в ответах есть номера без суффикса и варианты с суффиксом (например `0159945776-0040` и `0159945776-0040-4`), записи стягиваются к более полному номеру, а самостоятельные отправления с разными суффиксами остаются раздельно. В деталях показывается таблица расходов по отправлению (комиссия, эквайринг, доставка/логистика, обработка, продвижение, прочее), себестоимость по артикулам и итоговые маржа/маржинальность. Навигация в шапке включает ссылку «Начисления» для быстрого перехода между ценами, акциями, заказами и начислениями.

25. Страница `dashboard.html` собирает по выбранным магазинам и датам все заказы (FBS и FBO), тянет комиссии/себестоимости через `/v5/product/info/prices`, считает сумму заказов, выручку, маржу и маржинальность по дням и строит один комбинированный график: столбцы показывают суммарную выручку и маржу по всем выбранным магазинам, отдельные линии — выручку каждого магазина, линия «Выручка (все магазины), линия» даёт общий тренд, а линия «Маржинальность, %» показывает долю прибыли. Легенда позволяет включать/выключать любые серии, сверху график, снизу — таблица итогов по дням; магазины выбираются все сразу по умолчанию.

26. Страница `sync.html` автоматизирует обновление цен и остатков: читает магазины из `stores_config.json` (или sample/демо), тянет прайс поставщика по нужному складу, сопоставляет артикула с OZON (по `/v3/product/list`), подтягивает комиссии/логистику через `/v5/product/info/prices`, рассчитывает цену/минимальную цену по тем же формулам, что на странице цен (с учётом эквайринга 1,9 %, продвижения 1 %, логистики, доставки и упаковки 40 ₽), заполняет `old_price = price/0.6` и отправляет обновления цен (`/v1/product/import/prices`) и остатков (`/v2/products/stocks`). Динамическая наценка задаётся диапазонами (до 100 ₽, 300 ₽ и т. д.) и сохраняется в браузере; доступно автообновление 3 раза в день.

⚠️ Для страницы цен по‑прежнему нужен доступ к API OZON из браузера. Если CORS или корпоративная политика запрещает прямые вызовы, запустите `index.html` через небольшой прокси‑сервер либо перенесите запросы на backend.
